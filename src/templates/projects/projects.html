XIP

XIP is a maze solver with long-term planning that me and some friends have been developing for a few years now. This project is used in the Ciber-Mouse competition, a contest organized by the University of Aveiro, Portugal that is quite interesting.

We got the first place in 2006, second in 2008 and fourth in 2005, so this is a particularly successful project. Lots of technology and algorithms involved so I'll try to sum it up: C++ and SDL for the code and runtime debugging tool; A* for path planning; Fuzzy logic for short term decision such as avoiding colision; Morphological operations for knowledge consolidation

We also developed some satelite tools like a map editor and a log viewer in flash. The project has a blog with detailed information about this and more but, unfortunely for non-portuguese speakers, it's in portuguese.

Open Data Warehouse

While at Critical Software I worked on a project called ADW that had a very interesting approach to a build a scalable, distributed and heterogeneous data warehouse but it used horrible (as in acient) technology. I'm considering to implement it as a middleware in python. If you can dig them, there are a few papers about it.

The main goal of the project is to develop a technology that allows a dramatic reduction of the hardware, software, and administration cost when compared to tratidional data warehouses. The project's approach is based on the clever combination of two simple ideas: 1) uniform data striping to partition the data warehouse facts over an arbitrary number of computers, in such a way that queries can be executed in a true parallel fashion (a query is actually split into many partial queries), and 2) an approximate query answering strategy to deal with the momentary unavailability of one or more computers in the cluster. The ADW approach takes advantage of the specific characteristics of star schemas and typical data warehouse queries profile to guarantee optimal load balance of query execution and assures high scalability. In data warehouse striping the fact tables are distributed by an arbitrary number of computers and the queries are executed in parallel by all the computers, guarantying a nearly optimal speed up.
